<analysis_framework>
Analyze each category below. For each confirmed finding, provide evidence from the data.

### 1. Information Disclosure
- Server version headers (Apache/1.3, Apache-Coyote, nginx x.x.x)
- Directory listing enabled
- Stack traces, error pages, debug endpoints
- Internal paths exposed (/server-status, /manager/html, /actuator, /.git, /.env)
- Comments in HTML revealing sensitive info

### 2. Security Misconfiguration
- Missing security headers (Strict-Transport-Security, X-Frame-Options, X-Content-Type-Options, Content-Security-Policy)
- CORS misconfiguration (Access-Control-Allow-Origin: *)
- Default credentials on admin interfaces
- Exposed management consoles (Tomcat Manager, Jenkins, PHPMyAdmin)
- Directory with 302 redirects suggesting redirect-based access control

### 3. Injection Vulnerabilities
- SQL injection in URL parameters or form fields
- Cross-Site Scripting (XSS) in reflected or stored contexts
- Command injection in parameters
- Server-Side Request Forgery (SSRF) indicators
- Local/Remote File Inclusion (LFI/RFI)
- Template Injection (SSTI)

### 4. Authentication & Authorization
- Login pages without rate limiting
- Password reset functionality without proper tokens
- Session tokens in URLs
- Insecure authentication protocols
- Privilege escalation paths via forced browsing

### 5. Transport Security
- Missing HTTPS strict transport security
- Mixed content warnings
- Weak SSL/TLS cipher suites
</analysis_framework>
