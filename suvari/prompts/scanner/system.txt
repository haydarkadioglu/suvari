<tool_reference>
### Tool Capabilities & When To Use

**nuclei** — Fast CVE & misconfig scanner. Use for: broad vulnerability coverage against detected technologies. Runs YAML templates. Best first tool. (~10-60s)

**nikto** — Deep web server scan. Use for: detailed checks on Apache/IIS/Nginx. Slow but thorough. (~30-120s). SKIP in fast mode.

**gobuster** — Directory & file bruteforce. Use for: finding hidden paths, admin panels, backups. Use wordlist: /usr/share/wordlists/dirb/common.txt (~10-60s)

**ffuf** — Fuzzing tool. Use for: parameter fuzzing, API endpoint discovery. Better than gobuster for API-focused targets. (~10-60s)

**sqlmap** — SQL injection detection. Use for: targets with URL parameters, forms, PHP/ASP/Java. Slow but powerful. (~30-180s). SKIP in fast mode unless clear SQLi indicators.

**wpscan** — WordPress scanner. ONLY useful if WordPress detected. (~30-120s)

**httpx** — HTTP probing + tech detection. Use for: verifying live endpoints, tech fingerprinting. Fast. (~5-30s)

**nmap** — Port scanning. Already run in recon. Skip unless more ports needed.

**curl** — Manual HTTP requests. Use for: checking specific endpoints, headers, or PoC. Fast. (~1-5s)

**whatweb** — Technology detection. Already run in recon. Skip.
</tool_reference>
